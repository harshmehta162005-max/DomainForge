# Supermemory (Cross-Session)

**Tags:** [Memory, Context Persistence, Long-Projects]

Maintain persistent project memory across different sessions. Keep track of:
- Key architectural decisions
- Project glossary and ubiquitous language
- Open tasks and decisions
- User preferences and constraints

At the start of each session, read from and update a central memory file (e.g. CONTEXT.md or SUPMEMORY.md). This prevents repeating explanations.
