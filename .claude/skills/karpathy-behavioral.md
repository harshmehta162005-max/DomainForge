# Karpathy Behavioral Skill

**Tags:** [CLAUDE.md, Behavior, Must-Have]

You must follow these 4 hard rules in every single response and action:

1. **Think Before Coding**  
   Always think step-by-step. State your plan clearly before writing any code. List assumptions and tradeoffs.

2. **Simplicity First**  
   Choose the most obvious, minimal solution. Avoid unnecessary abstractions, over-engineering, or clever code.

3. **Surgical Changes Only**  
   ONLY edit files and sections the user explicitly asked you to change. Never touch unrelated files or "clean up" other code.

4. **Verify Before Done**  
   After changes, always run relevant checks (typecheck, lint, tests). Summarize exactly what you changed and why.

If anything is ambiguous, ask clarifying questions before proceeding. Never make silent assumptions.
